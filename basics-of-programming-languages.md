# Basics of programming languages

## [변수 (Variable)](https://github.com/cherrycoder9/cs-keywords/blob/main/basics-of-programming-languages/variable.md)

- 데이터를 저장하고 참조하는 메모리 공간.
- 프로그램 실행 중 값 변경 가능함.
- 식별자로 접근함.

## 자료형 (Data Type)

- 변수에 저장된 데이터의 유형과 크기를 정의.
- 값의 해석 방식과 허용 연산 결정함.
- 메모리 효율적 사용에 중요함.

## 리터럴 (Literal)

- 소스 코드에 직접 작성된 고정된 데이터 값.
- 자료형에 맞는 표기법 사용함.
- 변수와 달리 변경 불가함.

## 연산자 (Operator)

- 피연산자에 대해 정의된 연산을 수행하는 기호.
- 연산 종류와 피연산자 개수에 따라 분류함.
- 연산 우선순위 고려해야 함.

## 제어문 (Control Structure)

- 프로그램 실행 흐름을 제어하는 구문.
- 조건에 따른 분기, 반복 수행 등 가능케 함.
- 논리적 오류 방지에 유용함.

## 함수 (Function)

- 독립적인 기능을 수행하는 코드 블록.
- 입력을 받아 출력을 생성함.
- 코드 중복 제거와 프로그램 구조화에 핵심적임.

## 재귀 함수 (Recursive Function)

- 자신을 다시 호출하는 함수.
- 문제를 동일한 형태의 보다 작은 문제로 분할 정복함.
- 기저 조건 설정이 필수적임.

## 배열 (Array)

- 같은 자료형 데이터를 연속된 메모리 공간에 저장하는 자료구조.
- 인덱스로 요소 접근.
- 데이터 집합 표현과 반복 처리에 유용.

## 객체 (Object)

- 데이터(속성)와 동작(메서드)을 하나로 묶은 독립적 실체.
- 클래스를 기반으로 생성된 인스턴스.
- 객체지향 패러다임의 기본 구성요소.

## 클래스 (Class)

- 객체 생성을 위한 속성과 메서드가 정의된 틀.
- 객체의 설계도 역할.
- 코드 재사용성과 유지보수성 향상.

## 접근 제어자 (Access Modifier)

- 클래스나 멤버의 접근 범위를 제한하는 키워드.
- 캡슐화와 정보 은닉 구현에 필수적.
- 코드 안정성 보장.

## 캡슐화 (Encapsulation)

- 데이터와 관련 메서드를 클래스 내부로 포장하는 것.
- 외부에서의 직접 접근을 차단.
- 모듈화 및 응집도 증진.

## 정보 은닉 (Information Hiding)

- 객체 내부 구현을 외부로부터 숨기는 원칙.
- 불필요한 세부사항 노출 방지.
- 시스템 복잡도 경감.

## 상속 (Inheritance)

- 상위 클래스의 속성과 메서드를 하위 클래스에게 물려주는 것.
- 계층 구조 형성.
- 코드 중복 제거와 다형성 실현.

## 오버라이딩 (Overriding)

- 자식 클래스에서 부모 클래스의 메서드를 재정의.
- 상속받은 메서드의 동작을 변경함.
- 다형성 구현의 핵심 메커니즘.

## 다형성 (Polymorphism)

- 객체의 타입에 따라 동일한 메서드가 다르게 동작하는 특성.
- 코드의 유연성과 확장성 향상.
- 인터페이스와 상속을 통해 구현됨.

## 오버로딩 (Overloading)

- 같은 이름의 메서드를 매개변수만 다르게 여러 번 정의.
- 다양한 자료형과 인자 개수 처리 가능.
- 메서드 이름 재사용으로 코드 간결화.

## 추상화 (Abstraction)

- 객체의 불필요한 세부 사항을 숨기고 필수적인 특징만 표현.
- 복잡성 관리 및 코드 가독성 개선.
- 인터페이스와 추상 클래스로 구현됨.

## 인터페이스 (Interface)

- 구현 클래스가 반드시 구현해야 할 메서드 시그니처 정의.
- 다중 상속 지원 및 객체 간 통신 규약 제공.
- 느슨한 결합도 유지에 도움됨.

## 추상 클래스 (Abstract Class)

- 인스턴스화 불가능한 불완전한 클래스.
- 공통 속성과 동작 정의 후 구체적인 구현은 자식 클래스에 위임함.
- 코드 중복 최소화에 유용.

## 예외 처리 (Exception Handling)

- 런타임 오류 발생 시 프로그램 비정상 종료 방지를 위한 메커니즘.
- try-catch-finally 구문으로 구성됨.
- 에러 복구 및 사후 처리 수행.

## 제네릭 (Generics)

- 자료형을 파라미터화하여 재사용 가능한 코드 작성.
- 컴파일 시점에 타입 검사 수행.
- 불필요한 타입 변환 제거.

## 컬렉션 (Collection Framework)

- 데이터 그룹을 표현하고 조작하는 통일된 아키텍처.
- 상호 운용성과 일관성 있는 API 제공.
- 다양한 자료구조 구현체 포함.

## 자료구조 (Data Structure)

- 데이터 조직, 관리, 저장 방식을 정의.
- 효율적인 접근과 수정을 위한 알고리즘 제공.
- 문제 해결 복잡도 결정에 큰 영향.

## 람다 표현식 (Lambda Expression)

- 함수형 인터페이스를 구현한 익명 객체 생성 문법.
- 매개변수, 화살표, 함수 본문으로 구성.
- 코드 간결성과 가독성 향상.

## 스트림 (Stream)

- 데이터 처리를 위한 함수형 프로그래밍 스타일 인터페이스.
- 내부 반복을 통해 병렬 처리 가능.
- 지연 연산으로 성능 최적화.

## 포인터 (Pointer)

- 다른 변수, 상수, 함수의 메모리 주소를 저장하는 변수.
- 동적 메모리 할당과 참조에 사용.
- 잘못된 사용 시 위험할 수 있음.

## 스코프 (Scope)

- 프로그램 내에서 식별자가 유효한 영역.
- 변수 생존 기간과 접근성 결정.
- 스코프에 따라 식별자 해석이 달라짐.

## 상수 (Constant)

- 프로그램 실행 중 값 변경이 불가능한 데이터.
- 변경 불가능성으로 안정성 높임.
- 유지보수성 향상에 기여.

## 열거형 (Enumeration, Enum)

- 유한한 값들의 집합을 표현하는 자료형.
- 가능한 값을 명시적으로 나열함.
- 코드 가독성과 안전성 향상.

## 가비지 컬렉션 (Garbage Collection, GC)

- 동적 할당된 메모리 중 불필요해진 객체를 자동 탐지하여 해제함.
- 메모리 관리 부담 경감.
- 메모리 오류 가능성 감소.

## 네임스페이스 (Namespace)

- 식별자의 유효 범위를 제한하는 논리적 구조.
- 이름 충돌 방지와 코드 모듈화 지원.
- 대규모 프로젝트 관리 용이.

## 모듈 (Module)

- 특정 기능을 제공하는 코드 묶음.
- 캡슐화와 추상화 구현.
- 개발 및 유지보수 효율성 제고.

## 라이브러리 (Library)

- 재사용 가능한 코드의 집합.
- 공통 기능 제공으로 개발 시간 단축.
- 검증된 코드로 안정성 보장.

## API (Application Programming Interface)

- 소프트웨어 컴포넌트 간 상호작용을 위한 명세.
- 모듈 간 통신 방법과 데이터 교환 방식 정의.
- 시스템 확장성과 유연성 향상.

## 커링 (Currying)

- 다인수 함수를 일련의 단일 인수 함수로 변환하는 기법.
- 부분 적용 함수 생성 가능케 함.
- 함수형 프로그래밍의 핵심 개념.

## 클로저 (Closure)

- 함수가 선언된 환경의 변수를 캡처하여 해당 환경 외부에서도 접근 가능케 함.
- 상태 은닉과 모듈화 구현에 활용됨.

## 익명 함수 (Anonymous Function)

- 식별자 없이 정의되는 함수.
- 함수를 값처럼 다룰 수 있게 함.
- 고차 함수와 콜백 패턴 구현에 유용함.

## 매크로 (Macro)

- 소스 코드를 프로그램으로 처리하는 기능.
- 반복 코드 제거와 도메인 특화 언어 구현에 사용됨.
- 오용 시 가독성 저하 우려 있음.

## 이터레이터 (Iterator)

- 집합체의 요소를 순차적으로 접근하는 수단 제공.
- 내부 구조 노출 없이 일관된 방식으로 요소 탐색 가능케 함.

## 제네릭 프로그래밍 (Generic Programming)

- 타입에 독립적인 알고리즘과 데이터 구조를 작성하는 기법.
- 코드 중복 최소화와 타입 안전성 보장.
- 템플릿 또는 제네릭 문법 사용.

## 타입 추론 (Type Inference)

- 컴파일러가 문맥을 분석하여 변수나 표현식의 타입을 추론하는 기능.
- 명시적 타입 선언 없이도 타입 안전성 유지.
- 코드 간결성 향상.

## null

- 객체 참조가 어떤 객체도 가리키지 않음을 나타내는 특수 값.
- 초기화되지 않은 참조나 유효하지 않은 상태 표현.
- 널 검사를 통해 예외 방지.

## void

- 반환 값이 없는 함수의 반환 타입.
- 함수가 수행 결과를 돌려주지 않음을 명시.
- 프로시저 또는 부수 효과만 있는 함수에 사용.

## 값 타입 (Value Type)

- 변수에 값 자체를 저장하는 데이터 타입.
- 할당 또는 전달 시 값이 복사됨.
- 스택 메모리에 할당.
- 구조체, 열거형 등이 해당.

## 참조 타입 (Reference Type)

- 데이터에 대한 참조를 저장하는 데이터 타입.
- 할당 또는 전달 시 참조만 복사됨.
- 힙 메모리에 할당.
- 클래스, 인터페이스, 배열 등이 해당.
